<<TMF_Process>>
The objective of Probe Root Cause Of Failure process is to find out a root cause of failure causing resource trouble for dealing with hardware repair and  software upgrade.  The process is performed after resource healing, and analyzes the problem (e.g., the failure software bugs and/or hardware faults causing failure of network function). <br/>The responsibilities of these processes include:<br/>-Perform testing of resource for isolation of cause of failure .<br/>-Identify a real cause of failure.<br/>-Obtain existing knowledge information related to recovery.<br/>-Determine a optimal way to repair.<br/>-Determine schedule for repair work.<br/>-Update knowledge information coping with recovery if the event is unknown failure.<br/>
