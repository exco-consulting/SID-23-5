<<TMF_AF>>
Resource Trouble Analysis Aggregate Function permits, to identify <br/>• the trouble causes – that is to say the reasons why the installed resource is in trouble and among these reasons the root cause of the resource trouble <br/>• and the necessary resolution actions to solve the problem.<br/><br/>It is based on diagnostic actions (Test, inventories and historic consultation, workforce intervention…) and rules defined by Resource Problem Strategy and Policy Management. <br/><br/>The identification of the resource trouble cause can be made through:<br/>• Analysis of associated Service Trouble Ticket Information (first diagnostic)<br/>• Analysis of associated Resource Trouble Ticket Information (first diagnostic)<br/>• Analysis of Resource alarm events (fault of performance data …)<br/>• Issuing resource tests, including one time tests or a series of tests over a period of time<br/>• Analysis of previous resource test results, resource usage historic, QoS historic, equipment logs, anomaly historics, … <br/><br/>Remark: If the diagnostic action is a Resource Order, this will be created via the Resource Order Initialization function.<br/>
