<<TMF_AF>>
The Resource Specification Development Aggregate Function enables to create and update the catalogue at Resource level, for all types of resources (Resource Specification):<br/>• Physical Resources (ex: cable, passive equipment, …)<br/>• Logical Resources (ex: phone number, IP address, APPS such as TV, Maps, …)<br/>• Compound Resources (ex: DSLAM, box, …)<br/>Examples of Resource Specification:<br/>• iphone 5S and its characteristics such as colors available, size and weight, capacity, …<br/>• DSLAM Alcatel XXX and its characteristics such as size and weight, electricity consumption, number of DSL ports, …<br/><br/>It manages the links with the Supplier/Partner Product Catalogue for Resources specification that correspond to equipment rented to a supplier (devices, network equipments, hardware & software, …).<br/>It also manages the links to the Skill catalogue to identify for each type of resource – or of operation on a resource – which type of skill is necessary to make the intervention.<br/><br/>It also permits to describe in a short way resources types that as a Service Provider we don’t own or commercialize (ex: mobile phones used by our customers).<br/>
