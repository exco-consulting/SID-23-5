<<TMF_AF>>
The Service Availability Aggregate Function validates that the service or services specified on the Service Order, or necessary regarding the product items specified in the Customer Order are:<br/>• available at the specified customer/service location <br/>• feasible from a network and/or service platforms point of view<br/>• compatible with related existing services – with or without changing their characteristics or technical solution<br/>It includes Technical Feasibility (line type characteristics, coverage, equipment features …), and Technical Compatibility with the resources configurations already existing to support current customer installed services (refer to the Resource Availability function).<br/><br/>It can identify the need <br/>• for changing the current technical solution to respond to the order – according to allowed operations declared at catalogue level and engineering rules (Example: for an existing VOIP service, change H323 solution to SIP solution to add a new service such as Stop Secret)<br/>• for modifying existing installed Products characteristics to maintain global integrity of a set of inter-depending services (Example: when changing the installed address of a fixed access, it can be necessary to change the call number of the related voice service, if it depends on geography)<br/>• for ordering pre-requisite customer equipment (Example: the availability for broadband access is confirmed but the only possible – or preferred – solution is fiber, and it’s necessary to install an ONT and to simultaneously order a compatible box)<br/><br/>It also determines the standard delivery time needed, taking into account <br/>• resources availability<br/>• the need of human intervention at any point of the network – and time and skills necessary to plan it<br/>If an intervention is needed at the delivery address, regarding network context, it specifies it to the customer order capture, to allow it to manage an appointment.<br/><br/>To be noticed: if an intervention at the delivery address is always mandatory (or never necessary) for an ordered product, this information can be managed directly at product catalogue level.<br/>
