Resource Trouble Diagnostic Design Function defines diagnostic actions to identify the resource problem cause. It includes:<br/><ul>
<li>Diagnostic actions and rules to identify the resource trouble. This step permits to identify the installed resource(s) at fault</li></ul>
   - For example: simple resource test or resource test scenario, inventories consultation to identify installed resource and expected configuration characteristics, test historic consultation, etc.<br/><ul>
<li>Diagnostic actions  and rules to identify the resource trouble cause(s) and the possible root cause . This step permits to identify why the related installed resource(s) is (are) at fault.</li></ul>
   - For example: Plan workforce intervention to check an equipment, launch resource test scenario, query QoS historic, followed by resource test launch if the installed resource has a decreasing QoS, query resource usage historic, equipment logs, anomaly historic, etc.<br/>
