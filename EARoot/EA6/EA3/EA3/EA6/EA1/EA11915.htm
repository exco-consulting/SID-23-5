Product Test Execution Function includes:<br/><ul>
<li>Carrying out test priority: for example, the priority of a self-care test for a user is higher than a back-office test</li><li>Retrieval of appropriate inventory data and setting up the test context and configuration</li><li>Test execution with access to the various data required to perform product testing</li></ul>
<ul>
<li>A product test main actions will be to trigger service or resource tests depending on the product composition</li><li>Data collected during product test can be inventory data (for example in IS inventory) or data coming from service or resource tests</li><li>Test execution produces a test result that may contain a raw result and an enrichment of this result depending on the product test definition and associated rules.</li></ul>
     For example: for an access product test the raw result can be the associated bandwidth and an enrichment can be done depending on the user access type (fiber, xdsl) to evaluate if the result is suitable or not.<br/>
