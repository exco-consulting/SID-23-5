EmployeePayComponent is where an Employee's components of pay is defined. Each section of their pay is defined as an individual line item and can have different frequencies or currencies.<br/>This entity includes effective dated history.  Pay Component object stores all elements that are paid to employees like: base salary bonus target car allowance. <br/>For each pay component you need to define  <br/>• If the component is recurring or a one time payment <br/>• If recurring the frequency <br/>• If it is paid as an absolute amount or percentage <br/><font color="#ff0000">• If percentage what is it a percentage of (other Pay Component or Pay Component Group </font><br/>• If it is an earning or a deduction <br/>• If it is actual pay or target <br/>• If it is taxable or non-taxable <br/>• If it should be visible in Self Service <br/>• If it is used by compensation planning  Recurring pay components such as base salary should be defined with recurring attribute set to “Yes” and target attribute set to “No”. <br/>
