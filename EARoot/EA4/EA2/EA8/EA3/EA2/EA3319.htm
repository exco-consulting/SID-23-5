Figure SE.01 - Security Entity diagram
Examples of a Security Entity include devices, facilities, people, policies, circuits, networks, systems, and organizations.  <br/>Some aspect of the asset model is envisioned to be used by all parties engaged in network defense.  <br/>SecurityEntity has been sub-typed into three entities: <br/>        • ResourceSecurityEntity, which recognizes physical assets like facilities, equipment, systems, etc., <br/>        • PartySecurityEntity, which recognizes organizations, people, etc., and<br/>        • PartyRoleSecurityEntity, which recognizes that a Party can have one or more roles. <br/>Following SID convention, a SecurityEntity, which is derived from Entity, has one or more identification (EntityIdentification).<br/>
