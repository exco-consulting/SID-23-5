Figure ResCS.02 - Software Resource Specification Relationship diagram
There can be many types of relationships that exist among SoftwareResourceSpecifications.  These are shown in the Figure below.<br/>The SoftwareResourceSpecRelationship entity has purposely been made non-abstract so that new types of simple relationships can be defined in addition to those shown.  If base set of attributes in it are not sufficient then new subclasses can be explicitly modelled or the Characteristic pattern can be used to dynamically add the attributes to a new type.<br/>The SoftwareSpecRefinementRelationship subclass defines divisions of a SoftwareResourceSpecification into two or more other specifications.  Or fusions of two or more specifications into a single specification.  Or one specification can be substituted for another.  <br/>The SoftwareSpecConstraintRelationship subclass allows additional constraints to be defined for involved specification.  The SymmetricConstraint attribute of this subclass is used indicate if the relationship is from the perspective of just one specification or from both for example, A constrains B, and B constrains A.<br/>The SoftwareSpecAlternativeRelationship subclass defines specifications that are alternates of others.  Its attributes indicate if the alternate is an exact alternate.  The symmetric attribute indicates whether A is an alternate for B and B is an alternate for A, or A is an alternate for B only.<br/>
