Figure ResCS.08 - Model Association Enhancements - 1 diagram
As mentioned earlier, a number of associations have been enhanced and added to support the requirements of today’s digital world.  This section describes them.<br/>The Figure below shows one group of associations that were added/enhanced.<br/>The association between ResourceElement and InstalledSoftware deserves a few comments.  First, the association may appear to be redundant, because a CompoundResource is composed of Logical, Physical, and Compound Resources.  However, since ResourceElements can be realized purely by Software, this fact was deemed important enough to justify adding a direct association between the two entities.  Second, Figure CS.04 – Software Specification and Resource Specification shows a similar association between CompoundResourceSpecification and SoftwareSpecification.  The reason for the difference is that there is no ResourceElementSpecification entity.<br/>The InstalledSoftwareRuns (InstalledSoftware) association has been added to support scenarios such as an operating system running desktop software or a virtual machine operating system running a java virtual machine operating system.<br/>The addition of the association between Supplier and ResourceSpecification was mentioned in a previous section of this document.  It is shown here.<br/>The addition of the ResourceReferences association class was added to parallel the recursive association between ResourceSpecifications.  It enables one Resource to reference another for a variety of reasons, such as one Resource replacing or being an alternate of another or to model a hierarchy of Resources.<br/>
