<font color="#e0121d"><b>Computing and Software Resources Introduction</b></font><br/><font color="#29313b">The model components presented here demonstrates support for Computing and Software Resources.  A number of scenarios are used to demonstrate this support.  Note that Computing Resources are not explicitly modelled in the Information Framework (SID).  Support for them is shown via the scenarios and associated object (instance) diagrams.</font><br/><font color="#29313b"><br/></font><font color="#29313b"><b>Computing and Software Resource Requirements</b></font><br/><font color="#29313b">Described here are the requirements that drove the development of this part of the SID.</font><br/><ul>
<li><font color="#29313b">Support the entire software lifecycle processes from ordering through to running software. Some key items: </font></li></ul>
<ul>
<li><font color="#29313b">Software inventory (planning to actively installed and running) provided in a manner consistent with hardware inventory</font></li><li><font color="#29313b">Usage and performance management (already supported by the SID)</font></li><li><font color="#29313b">Fault management with the possible addition of enumerations to Probable Cause</font></li><li><font color="#29313b">Configuration, including dependencies <b>to</b> underlying resources and dependencies <b>from</b> applications that rely on given software, for example Application A or Function B may require Software Packages X, Y and Z in order to operate. </font></li><li><font color="#29313b">Activation and deactivation </font></li><li><font color="#29313b">Support running software functionality in a way that is consistent with and integrates with hardware functionality (OSI stack thinking).</font></li><li><font color="#29313b">Support the management of hardware-based resources <u>used by</u> software (CPU, memory, disk)</font></li></ul>
<ul>
<li><font color="#29313b">Do we want to consider management of the specific applications and functions supported by a given software instance? This is effectively Application management, for example the management of Software network elements</font></li><li><font color="#29313b">Allow for future support of data centers, including VMs.</font></li></ul>
<font color="#29313b">There are eTOM impacts implied here as it does not make much mention of Computing and Applications.</font><br/>
