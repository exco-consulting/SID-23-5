Figure ResCS.04 - Software Specification and Resource Specification diagram
The Figure below shows associations between SoftwareSpecification, CompoundResourceSpec, and ResourceSpecification.<br/>The first simple association is used to specify that a CompoundResourceSpecification may be realized by SoftwareSpecification alone and not a combination of at least one PhysicalResourceSpec and at least one LogicalResourceSpec.  This changes the definition of a CompoundResourceSpec and a CompoundResource that will be shown in an upcoming figure.<br/>The association class is used to show the status of a SoftwareSpecification that can run on any ResourceSpecification.  For example, the status may reflect that it can or can no longer be run.  The association was made so that any type of resource (within reason!) could run Software, such as a CompoundResource, LogicalDevice or PhysicalDevice.<br/>The SoftwareSpecificationUses association was added so that any type of Software could use any type of Resource (again, within reason).  For example, Software could use memory (logical or physical), memory databases, or physical disks of any type.<br/>HostingPlatformRequirementSpec has been introduced to represent the requirements that must be fulfilled by the platform on which the InstalledSoftware will run (e.g. compute, storage, network).<br/>The derived association “/SoftwareSpecificatonRequires” is used to document the requirements necessary to a platform to be capable of hosting the concerned software before it is installed.<br/>
