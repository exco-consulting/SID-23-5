<font color="#e0121d"><b>Overview of OMG SMM </b></font><br/><font color="#29313b">SMM was designed to specify metrics for software artifacts (programs, code, modules).  It arose from the Architecture Driven Modernization Task Force of OMG.  They are primarily concerned with analysis and characterization of software artifacts (e.g., programs, code libraries, software modules, etc.)</font><br/><font color="#29313b">As such it is a static model, with no concept of combining measurements across time.</font><br/><font color="#29313b">The assumption is that the system being analyzed is comprised by a set of elements, which are instances of a metamodel. This "target" metamodel defines classes and associations that express the structure of the system being analyzed by a SMM model.</font><br/><font color="#29313b">Figure A-1 shows the core concepts of SMM.</font><br/>
