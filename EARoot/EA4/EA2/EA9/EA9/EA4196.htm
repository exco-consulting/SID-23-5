Figure Me.09 - Metric Measure Notification definition diagram
Different kinds of consequences may be applied as a consequence of threshold. The following Figure shows how to describe notification consequences.<br/>A MetricDefMeasureConsequence either results in a prescribed action (prescribedAction) or one or more ConsequenceMetricNotificationSpecs.<br/>If a MetricMeasureConsequence is triggered by a MetricMeasure (through a MetricMeasureThresholfRule) according to a MetricDefMeasureConsequence and the MetricDefMeasureConsequence results in ConsequenceMetricNotificationSpecs, then a ConsequenceMetricNotification is created for each of them.<br/>A ConsequenceMetricConsequence is a type of ConsequenceMetricNotification, which in turn is a type of Notification.<br/>A ConsequenceMetricNotification is described by a ConsequenceMetricNotificationSpec.<br/><i>Note: To be consistent with the pattern Specifcation used for ConsequenceMetricNotificationSpec / ConsequenceMetricNotification, a NotificationSpecification should exist in the Business Interaction ABE. This concept would specify the content pattern for each corresponding notification.</i><br/>
