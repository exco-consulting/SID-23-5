Figure Me.03 - Metric Definition Description and Dimension diagram
The following Figure shows how a MetricDefinition’s description can be extended as well as dimensional attributes that define a Metric.<br/>The MetDefDescExtension entity enables additional descriptive text, such as references and owner notes, to be defined for a MetricDefinition. The metDefType attribute specifies the type of descriptive text contained in the descriptionExtension attribute.  <br/>The four-dimension related entities provide the ability to designate methods of organizing individual metrics into common sets, related by the value of the named dimension.  <br/>For example, a metric for revenue–individual instance values of revenue could be organized in the dimension of geography (Region 1 Revenues, Region 2 Revenues, etc.), or by Product (Wireless Revenue, Wire-line Revenue, Broadband Revenue, Advertising Revenue, etc.). <br/>The characteristic term when describing dimension is “by”, as in “Revenue, by Product” or “Revenue, by Region”.<br/>
