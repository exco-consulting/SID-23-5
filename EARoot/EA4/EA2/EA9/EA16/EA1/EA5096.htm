<<TMF_BusinessEntity>>
Represents the grouping of the Anomaly based on a particular need. For example, if the Anomaly need to be grouped based on a set of network nodes in a network topology or particular infrastructure deployment (rack/slot). AnomalyGrouping defines additional criteria for grouping the Anomaly management logic rather than limiting to individual entities. The grouping criteria are defined using AnomlayGroupingSpecification.<br/>
