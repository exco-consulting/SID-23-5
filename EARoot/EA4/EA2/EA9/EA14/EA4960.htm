Figure U.12 - IPDR as a Subclass of Resource Usage diagram
The second approach represents a modification of the current SID Usage model with the introduction of several relationships not currently modeled and explicit modeling of IPDR attributes not supported by the modified Usage model.  The resultant model is shown in Figure U.12 using the IPDR WAN Usage detail record as an example.  Each IPDR would be modeled explicitly as a subclass of the ResourceUsage entity.<br/>Note:  If this option is chosen then each new type of usage needs to be explicitly modeled, resulting in an unstable model.  It is not the preferred option.  The preferred option is shown figure U.10 and U.11.<br/>
