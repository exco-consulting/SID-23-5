A quality or distinctive feature that defines an entity. The characteristic can be take on a discrete value, such as color, can take on a range of values, (for example, sensitivity of 100-240 mV), or can be derived from a formula (for example, usage time (hrs) = 30 - talk time *3).<br/>

Notes: An indicator that specifies if a value is unique for the specification.<br/><br/>Possible values are: "unique while value is in effect", "unique whether value is in effect or not" and "not unique".<br/><br/>     - "unique while value is in effect" means that as long as an instance in the inventory has this value valid, no other instance can have the same value<br/><br/>     - "unique whether value is in effect or not" means that never anothe instance in the inventory can have the same value<br/><br/>     - "not unique" means that several instances of the Entity in the inventory might have the same value.<br/><br/><br/><br/>Specifies if the value of this characteristic is unique across all entities instantiated from the specification that uses this characteristc. For example, consider a ProductSpecification for a set-top box, with a CharacteristicSpecification cardID. Each set-top box must have a different value for cardID, so this isUnique attribute would be set to true for the characteristic.<br/>
