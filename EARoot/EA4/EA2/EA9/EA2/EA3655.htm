Figure R.19 - Identifying Entities Using the RootEntityType Entity diagram
The second option is shown in Figure EI.07 – Identifying Entities Using the RootEntityType Entity below. This option is used for Entities that don’t have an associated EntitySpecification, and can also be used in the first case if desired. This option introduces an entity (called RootEntityType) into the model whose instance is the business entity that is to be identified. Examples of business entities that use this approach include Location (identified by the 8-byte Network Site Format), Individual, Organization, Customer, Supplier, Market Segment and so forth.<br/>
