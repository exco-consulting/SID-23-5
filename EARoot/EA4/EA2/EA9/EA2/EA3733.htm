<font color="#29313b">An association has been added between the association role entities and their related root entities. Each association entity now references the entity that represents the role AND the related entity.  This should improve performance by providing the ability to access both entities involved in an association in a single database record (row). For example, consider the instance of the Base Station Controller AssociationRoleSpecification; this role is an AssociationRoleSpecification instance for the Mobile Switching Center “supports” Base Station Controller instance of AssociationSpecification; this instance of AssociationRoleSpecification would include the foreign key referencing the Base Station Controller RootEntityType instance, as well as secondary reference to the Mobile Switching Center RootEntityType instance. The associations were added to the UML model so that the applicable foreign keys would be automatically generated.</font><br/><font color="#29313b">Figure As.04b – Association Meta-Model Logical Data Model shows the logical data model with the associations labeled primary and secondary to reflect this.</font><br/>
