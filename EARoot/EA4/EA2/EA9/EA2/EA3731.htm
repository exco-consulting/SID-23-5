<font color="#e0121d"><b>Meta-Modeling Behavior</b></font><br/><font color="#29313b">The last component of the meta-model is the model for behavior, or operations/methods. This is beyond the scope of what is described here, as there are quite a few variations on how this could be modelled. And, any model presented here may not be a suitable representation in many cases. However, a starting point would be to use the properties of operations in the UML model as a set of requirements.  </font><br/><font color="#29313b">Besides these, there would also have to be provisions in the meta-model on how to invoke the behavior. One other implementation consideration is that there should be a separate library for the code, since this code would typically be developed outside of a solution. For example, if a solution vendor provides this capability as part of a solution, the responsibility for this additional code should not be theirs.</font><br/><font color="#e0121d"><b>The Meta-Model Database</b></font><br/><font color="#29313b">The meta-model database described in this section represents a simplification of the UML model. It was designed with performance considerations in mind. There are certainly other implementations of the meta-model that may or may not perform better.</font><br/><font color="#29313b">The model of the database supporting the UML model includes two associations that can be added to the information model or the physical data model. The model that is shown here is based on associations added to the UML model shown in Figure As.04a – UML Association Implementation Model.</font><br/>
