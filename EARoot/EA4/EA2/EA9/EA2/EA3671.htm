Root Business Entities/Characteristic ABE Illustration diagram
<font color="#29313b">Figure Ch.01 – Characteristic Specification depicts the basic CharacteristicSpecification pattern.  The pattern enables attributes unique to a type of entity to be created via its association to EntitySpecification and enables attributes unique to a certain entity to be created via its association to RootEntityType.  A RootEntityType is an entity in the SID.  For example, characteristics could be defined for entities (instances of RootEntityType) such as Product, Customer, and ProductSpecification that apply to all instances of those entities. Instance diagrams are shown in Figure Ch.03 – Characteristic Spec Instance – Example 1 and Figure Ch.04 – Characteristic Spec Instance – Example 2. Definitions of entities and attributes are provided in the data dictionary at the end of this document.</font><br/><font color="#29313b">An example of the use of the pattern will also be illustrated using the following figure.</font><br/><font color="#29313b">The EntitySpecification, which could be a ProductSpecification, instance is for a Linksys USB Ethernet Cable Modem DOCSIS.  Each tab, Features, Description, Additional Info, and Shipping Info is an instance of CharacteristicSpecification.  The Features CharacteristicSpec is a composite that is comprised of an instance of CharacteristicSpec for each feature line.  Instances of the CharacteristicSpecRelationship are used to compose the Features of the instances of feature line.  </font><br/><font color="#29313b">In this example, the CharacteristicSpecValue and CharSpecValueRelationship entities are not used. An example of their use could be for a set of coordinated colors for the modem.  For example, if a choice for a faceplate color (CharacteristicSpec) is blue (CharacteristicSpecValue), then the only color for the case color is black.  This rule would be implemented by an instance of CharSpecValueRelationship that relates the color blue with the color black with a value for charValueRelationshipType of “inclusive”.  </font><br/><font color="#29313b">See an example in Annex: More Details on Characteristics.</font><br/><br/>
