As well as being having names for Parties, we also need more formal ways of identification [Coad Letter]. When a customer applies for a service, for instance, they may need to provide some proof of identity. The Party Identification entities allow for this information to be stored and accessed as required.<br/>Note that the Party Identification structure has been defined in a similar manner to Party Name.<br/>Concrete subclasses to Individual Identification and Organization Identification should be created as required. Figure 3 shows some common example concrete entities.<br/>The Party Identification Specification entity may be implemented as a Policy, using a Rules Engine or via a set of Templates that detail a valid set of Identification entries sufficient for a given purpose.<br/>
